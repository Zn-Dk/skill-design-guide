# 自反哺闭环

> 📌 **适用范围**：Standard / Heavy 档可选特性。Lite 档通常不需要此机制。
>
> 当 Skill 需要随使用持续进化时，引入自反哺闭环。纯指引型 Skill 可跳过本章。

---

## 闭环流程总览

```
用户纠正 → CORRECTIONS.md 记录 → 周期扫描 → 固化到规则文件
                                                ↓
                          LEARNINGS.md ← 根因分析 + 优化待办
```

---

## 6.1 实时记录（Agent 硬约束）

当用户明确纠正 Agent 输出时，Agent **必须**：

1. 记录上下文（触发场景 + 错误输出 + 正确答案）
2. 追加到 `.learnings/CORRECTIONS.md`
3. 标记 `已固化: ❌`

### CORRECTIONS.md 条目格式

```markdown
## [日期] 纠正 #N

**触发语句**：{用户原话}
**原始输出**：{Agent 给出的错误结论}
**正确答案**：{用户纠正后的正确信息}
**涉及模块**：{相关模块/阶段}
**已固化**：❌
```

---

## 6.2 周期固化

定时或手动扫描未固化记录，将经验提炼为规则：

| 步骤 | 动作 | 产出 |
|------|------|------|
| 1 | 扫描 `已固化: ❌` 的条目 | 待处理列表 |
| 2 | 提取模式规律 | 规则候选 |
| 3 | 更新规则文件（modules/ 或 assets/） | 新增/修改规则 |
| 4 | 记录优化项到 `LEARNINGS.md` | 改进追踪 |
| 5 | 标记 `已固化: ✅` | 闭环完成 |

### 固化判断标准

- **同类纠正 ≥ 2 次** → 必须固化为规则
- **单次但影响面大** → 建议固化
- **单次且为个例** → 记录观察，暂不固化

---

## 6.3 LEARNINGS.md

记录已知问题和改进方向，供后续迭代参考：

```markdown
## [优先级] 问题标题

**状态**：待修复 | 已完成
**根因**：{问题根本原因}
**改进方案**：{具体优化措施}
**关联纠正**：CORRECTIONS.md #N
```

---

## 何时引入此机制

| 条件 | 建议 |
|------|------|
| Skill 产出有"对错"之分（如数据分析、诊断） | ✅ 强烈建议 |
| Skill 规则会随项目演进变化（如 Code Review） | ✅ 建议 |
| Skill 是静态指引（如编码规范、样式指南） | ❌ 不需要 |

## 设计意图

自反哺机制确保 Skill 是一个**持续进化的系统**，而非静态文档：

1. **短期**：用户纠正立即被记录，避免同一错误重复出现
2. **中期**：周期固化将个案经验提炼为通用规则
3. **长期**：知识库持续增长，Skill 的准确率随使用量提升
